## 화이트리스트 적용 규칙

### 1. 정확한 도메인 일치
- SAFE: 위 목록과 정확히 일치하는 도메인
- DANGEROUS: 타이포스쿼팅 변형 (예: naverr.com, kakaoo.com, kbstar-event.com)

### 2. 서브도메인 처리
- 알려진 서브도메인 (예: nid.naver.com, mail.google.com, tworld.co.kr): SAFE
- 의심스러운 서브도메인 (예: login-verify.naver.com, security-alert.kakao.com): SUSPICIOUS 이상
- 복잡한 서브도메인 (예: abc-def-ghi.naver.com): DANGEROUS

### 3. 프로토콜 및 경로 검증
- HTTPS 사용 권장, HTTP 사용 시 주의 필요
- 비정상적인 경로 (예: /verify-account-urgent/): 추가 검증 필요

### 4. 하이픈(-) 및 숫자 사용 주의
- 공식 도메인에 없는 하이픈이나 숫자 추가: 타이포스쿼팅 의심
- 예: kb-star.com, kbstar2.com → DANGEROUS

### 5. 최종 판단 우선순위
1. 블랙리스트 매칭 → 즉시 DANGEROUS
2. 화이트리스트 정확 매칭 → 기본 SAFE (단, 서브도메인/경로 재검토)
3. 타이포스쿼팅 패턴 → DANGEROUS
4. 의심 패턴 감지 → SUSPICIOUS
5. 알 수 없는 도메인 + 위험 패턴 → DANGEROUS
